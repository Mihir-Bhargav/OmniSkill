# /clarify

Rewrite the text in this conversation so it's easy to understand — remove jargon and complexity without dumbing it down.

How:
- Replace technical terms with plain equivalents, or explain them in brackets on first use
- Break complex sentences into shorter ones
- Use active voice instead of passive ("the team decided" not "a decision was made")
- Replace abstract nouns with concrete ones ("improvement" → "faster load times")
- Cut unnecessary qualifications ("somewhat", "rather", "it could be argued that")

The goal is that someone unfamiliar with the subject can follow the text without losing the key ideas.

If a technical term has no good plain equivalent and the audience needs it, keep it but explain it once.

Do not:
- Add examples or analogies unless the text is genuinely unclear without them
- Change the structure or argument

Return the clarified version only.
