# /analogy

Explain the concept in this conversation using a concrete analogy.

How to do this well:
1. Identify the core mechanism of the concept — what is the essential thing it does or how it works?
2. Find something familiar that works the same way — an everyday object, a physical process, a common experience
3. Map the analogy explicitly — "X in the concept is like Y in the analogy because..."
4. State where the analogy breaks down — every analogy has limits; name them so the person doesn't over-apply it

Rules:
- One good analogy beats three mediocre ones. Pick your best and commit to it
- The analogy should make the concept click, not just sound clever
- Use something the specific person is likely to know — if context suggests a technical audience, a technical analogy is fine; if general audience, keep it everyday
- Don't use analogies for things that are already simple — only use them where the concept is genuinely hard to grasp
- After the analogy, test it: "Does this help? Want me to try a different angle?"
